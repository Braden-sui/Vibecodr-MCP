# 12. OpenAI Submission Dossier Plan

## Purpose

Capture all data required for ChatGPT App submission and internal readiness review in one place.

## Submission Metadata Checklist

- App name
- App description
- Company or publisher legal name
- Company website
- Support email
- Privacy policy URL
- Terms of service URL
- Category and use case summary
- Tool inventory and capability summary
- Authentication model description
- Production MCP endpoint URL
- Widget domain and CSP summary
- Test prompts and expected outcomes
- Screenshots and short demo narrative

## Required Technical Evidence

1. MCP endpoint evidence
- production URL
- /mcp reachability proof
- auth behavior proof

2. Tool descriptor evidence
- names and schemas
- impact hints
- read-only vs mutating classification

3. Widget evidence
- load and interaction screenshots
- tool result handling screenshot
- error handling screenshot

4. Policy and safety evidence
- data handling summary
- abuse prevention controls
- user support and escalation path

## Data Collection Plan

### Owner: Product and Engineering

- complete metadata fields
- finalize app copy
- approve category and public description

### Owner: Backend

- provide production endpoint details
- provide auth scheme details
- provide tool schema export

### Owner: Frontend

- provide widget screenshots and UX behavior notes
- provide responsive behavior evidence

### Owner: Security

- provide security control summary
- provide incident and abuse handling summary

## Review Packet Structure

1. App metadata
2. Architecture summary
3. Tool catalog
4. Authentication flow
5. Security and privacy summary
6. Test prompts and results
7. Screenshots

## Final Go and No-Go Criteria

- all required fields complete
- endpoint and auth verified in production
- test prompts pass consistently
- support contacts and policies publicly reachable
- submission packet reviewed and approved internally
